Sealing Off Holes in the Great Firewall

3 Apr 2015 in technology

Two weeks ago there was a hacker attack on GreatFire.org. A website which monitors content blocked by the Great Firewall and offers tools to subvert it.

A week later we heard about a large scale DDoS attack on GitHub. GitHub said:

Based on reports we've received, we believe the intent of this attack is to convince us to remove a specific class of content.

GreatFire.org itself has a GitHub page at github.com/greatfire with the source for their page and proxy servers. Apparently the DDoS attack was performed by hijacking web traffic for Baidu ads and redirecting it to GitHub. A more technical analysis of the attack can be found here, here and here.

All together an interesting piece of how China tries to seal off holes in the Great Firewall.

On a side note: some days ago I found a tool to easily set up your own proxy server. Kind of like a poor man's VPN. It's called sshuttle. I will test it next time I'm behind the Great Firewall.
